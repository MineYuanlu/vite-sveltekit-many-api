# Agent Notes

## Project Overview

A Vite plugin (`apiRoutes()`) that auto-generates SvelteKit API routes from `-api.server.ts` files. **Early stage** ŌĆö architecture and conventions are still evolving.

## Dual-Mode Generation

The plugin produces files in two patterns:

- **Per-endpoint companions**: `+server.ts` and `api.remote.ts` (one per API file)
- **Aggregated registries**: `openapi-registry.server.ts`, `mcp-registry.server.ts` (one for all endpoints)

New generators can be added under `src/generators/` (companions) or `src/generators/registry/` (registries).

## Architecture

| File | Role |
|------|------|
| `src/index.ts` | Plugin entry ŌĆö hooks for build (`buildStart`), dev (`configResolved` + `configureServer`), and CLI (`process.argv[2] === 'generate'`) |
| `src/scanner.ts` | Walks `src/routes/api/` for `-api.server.ts`, delegates to generators |
| `src/parser.ts` | Parses exports via `es-module-lexer` ŌĆö detects `METHOD`, `zMETHOD`, `nMETHOD`, `dMETHOD` |
| `src/watcher.ts` | Dev-mode file watching ŌĆö per-file changes trigger companion regeneration immediately; registries update with 50ms debounce |
| `src/generators/server-file.ts` | Generates `+server.ts` (import handlers, call `$api/common.server` helpers) |
| `src/generators/remote-file.ts` | Generates `api.remote.ts` (wraps handlers with `$app/server` `query`/`command`) |
| `src/generators/registry/openapi.ts` | Generates `openapi-registry.server.ts` |
| `src/generators/registry/mcp.ts` | Generates `mcp-registry.server.ts` |

## Critical Conventions

### ESM Import Paths
`tsconfig` uses `"module": "NodeNext"`. **Every relative import must end with `.js`**:
```typescript
import { foo } from './config.js';   // Ō£ģ
import { foo } from './config';      // ŌØī TS2835
```

### Directory Imports
`tsup`/`esbuild` cannot resolve directory imports without extension. Use explicit `index.js`:
```typescript
import { generateRegistryFiles } from './generators/registry/index.js';  // Ō£ģ
import { generateRegistryFiles } from './generators/registry';           // ŌØī
```

### Generated File Safety
All generated files start with `// @generated by @yuanlu_yl/vite-sveltekit-many-api ŌĆö DO NOT EDIT\n` (see `config.ts`). The plugin **skips** files without this marker and will not overwrite them. Generated files are gitignored automatically.

### CLI Entry
Running `node -e "require('...')" generate` works because `src/index.ts` checks `process.argv[2] === 'generate'` at module level.

## Code Style

- Prettier: tabs, singleQuote, printWidth 120, trailingComma `all`
- TypeScript strict mode enabled (`noUnusedLocals`, `noUnusedParameters`, etc.)

## Key Constraints

- **No hard-coded path overrides yet** ŌĆö `API_ROUTES_DIR = 'src/routes/api'` and `API_NAME = '-api.server.ts'` are fixed in `config.ts`.
- **Parser is regex-based** ŌĆö `dMETHOD` description and `nMETHOD` value are extracted with regex from source text, not AST. Complex expressions may not be parsed correctly.
- **Tests are minimal** ŌĆö only one basic smoke test exists. New features should include tests.
- **`es-module-lexer` is a runtime dependency** ŌĆö used for fast export scanning; it must be bundled or available at runtime.

## When Modifying

- Adding a new generator? Update `scanner.ts` (if per-endpoint) or `generators/registry/index.ts` (if registry).
- Changing generated file format? Bump the marker or handle migration ŌĆö old generated files without the marker won't be touched.
- Changing constants in `config.ts`? Check all generators and `watcher.ts` ŌĆö paths and file names are referenced in multiple places.
