# Agent Notes

## Project Overview

A Vite plugin (`apiRoutes()`) that auto-generates SvelteKit API routes from `-api.server.ts` files. **Early stage** — architecture and conventions are still evolving.

## Dual-Mode Generation

The plugin produces files in two patterns:

- **Per-endpoint companions**: `+server.ts` and `api.remote.ts` (one per API file)
- **Aggregated registries**: `registry.server.ts`, optionally `registry.messages.ts` (one for all endpoints, in `apiDir` root)

New generators can be added under `src/generators/` (companions) or `src/generators/registry/` (registries).

## Architecture

| File                                 | Role                                                                                                                                  |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------- |
| `src/index.ts`                       | Plugin entry — hooks for build (`buildStart`), dev (`configResolved` + `configureServer`), and CLI (`process.argv[2] === 'generate'`) |
| `src/config.ts`                      | Constants, config interfaces (`ApiRoutesConfig`, `UtilConfig`, `GenerateConfig`, `MessagesConfig`), and defaults                      |
| `src/types.ts`                       | Shared types: `ApiMethodDef`, `MethodInfo`, `EndpointInfo`, `StandardSchemaV1`                                                        |
| `src/scanner.ts`                     | Walks `apiDir` for `-api.server.ts`, delegates to generators                                                                          |
| `src/parser.ts`                      | Parses exports via `es-module-lexer` — detects `METHOD`, `zMETHOD`, `nMETHOD`, `dMETHOD`                                              |
| `src/path-utils.ts`                  | `resolveRealPath`, `getRoutePath(filePath, apiDir)`, `getApiUrlPath(routePath)`                                                       |
| `src/file-writer.ts`                 | `writeIfChanged`, `removeGeneratedFile`, `syncEndpointGitignore`, `ensureUtilTemplate`                                                |
| `src/watcher.ts`                     | Dev-mode file watching — per-file changes trigger companion regeneration immediately; registries update with 50ms debounce            |
| `src/generators/server-file.ts`      | Generates `+server.ts` (import handlers, call `$api/common.server` helpers)                                                           |
| `src/generators/remote-file.ts`      | Generates `api.remote.ts` (wraps handlers with `$app/server` `query`/`command`)                                                       |
| `src/generators/registry/openapi.ts` | Generates `registry.server.ts` (`generateRegistry`) and `registry.messages.ts` (`generateRegistryMessages`)                           |
| `src/generators/registry/index.ts`   | Orchestrates registry generation, guards on `generate.registry` and `generate.messages`, manages `apiDir/.gitignore`                  |

## Critical Conventions

### ESM Import Paths

`tsconfig` uses `"module": "NodeNext"`. **Every relative import must end with `.js`**:

```typescript
import { foo } from './config.js'; // ✅
import { foo } from './config'; // ❌ TS2835
```

### Directory Imports

`tsup`/`esbuild` cannot resolve directory imports without extension. Use explicit `index.js`:

```typescript
import { generateRegistryFiles } from './generators/registry/index.js'; // ✅
import { generateRegistryFiles } from './generators/registry'; // ❌
```

### Generated File Safety

All generated files start with `// @generated by @yuanlu_yl/vite-sveltekit-many-api — DO NOT EDIT\n` (see `config.ts`). The plugin **skips** files without this marker and will not overwrite them. Generated files are gitignored automatically.

### CLI Entry

Running `node -e "require('...')" generate` works because `src/index.ts` checks `process.argv[2] === 'generate'` at module level. Supports `--config <file>` to specify a custom vite config path.

## Code Style

- Prettier: tabs, singleQuote, printWidth 120, trailingComma `all`
- TypeScript strict mode enabled (`noUnusedLocals`, `noUnusedParameters`, etc.)

## Key Constraints

- **`apiDir` is configurable** — defaults to `src/routes/api`, overridable via `ApiRoutesConfig.apiDir`. All generators accept it as a parameter; `getRoutePath` requires it to compute correct route paths.
- **`groupPattern` is configurable** — defaults to `/api/v[^/]+/([^/]+)`, overridable via `ApiRoutesConfig.groupPattern`. Used to extract group labels from API URLs in the registry.
- **Parser is regex-based** — `dMETHOD` description and `nMETHOD` value are extracted with regex from source text, not AST. Complex expressions may not be parsed correctly.
- **Tests are minimal** — only a basic smoke test suite exists. New features should include tests.
- **`es-module-lexer` is a runtime dependency** — used for fast export scanning; it must be bundled or available at runtime.
- **`StandardSchemaV1` from `@standard-schema/spec`** — schema type is framework-agnostic. `zMETHOD` can be any `StandardSchemaV1`-compatible schema (Zod, Valibot, etc.).

## Config Options

```typescript
apiRoutes({
	util: {
		path: 'src/lib/api/common.server.ts', // template copy detection path (null = skip)
		imp: '$lib/api/common.server', // import path used in generated +server.ts
		schema: 'standard' | 'zod', // which util template to copy
	},
	generate: {
		server: true, // generate +server.ts
		remote: true, // generate api.remote.ts
		registry: true, // generate registry.server.ts
		messages:
			false |
			{
				from: '$lib/paraglide/messages', // message module import path
				export: 'm', // imported object name
				returnType: 'string', // return type of message functions
				keyPrefix: 'group', // key prefix; actual key is '{keyPrefix}.{groupName}'
			},
	},
	apiDir: 'src/routes/api', // root directory to scan
	groupPattern: '/api/v[^/]+/([^/]+)', // regex to extract group from API URL
});
```

## When Modifying

- Adding a new companion generator? Update `scanner.ts` to call it alongside `processServerFile`/`processRemoteFile`.
- Adding a new registry generator? Update `generators/registry/index.ts`.
- Changing generated file format? Bump the marker or handle migration — old generated files without the marker won't be touched.
- Changing `apiDir` handling? It flows through: `index.ts` → `scanner.ts` → `processApiFile` → `processServerFile`/`processRemoteFile` → `getRoutePath(filePath, apiDir)`. Watcher also receives it directly.
